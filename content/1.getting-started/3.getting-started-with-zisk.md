This guide gets you from zero to a verified proof in one command.

**Prerequisite:** [Install ZisK](/getting-started/install-linux) and confirm it works:

```bash
cargo-zisk --version
```

## Create a project

Generate a new project using the `cargo-zisk sdk new` command:

```bash
cargo-zisk sdk new sha_hasher
cd sha_hasher
```

This creates a workspace with two crates:

```
sha_hasher/
├── Cargo.toml          # workspace root
├── guest/
│   ├── Cargo.toml
│   └── src/main.rs     # the program you're proving
└── host/
    ├── Cargo.toml
    ├── build.rs        # compiles the guest + generates default input
    ├── src/main.rs     # the host: runs the guest, generates proof, verifies
    └── bin/            # optional single-stage binaries
        ├── execute.rs
        ├── prove.rs
        ├── compressed.rs
        ├── plonk.rs
        ├── verify-constraints.rs
        └── ziskemu.rs
```

The template comes with a SHA-256 example that hashes an input `n` times. The **guest** runs inside the ZisK VM and produces a provable result. The **host** (`host/src/main.rs`) orchestrates everything: it feeds input, runs the guest, generates a proof, and verifies it.

---

## Run it

One command does everything:

```bash
cargo run --release
```

This builds the guest, executes it, generates a VADCOP STARK proof, and verifies it. You should see output ending with:

```
<<< EXECUTE (226ms)
>>> CALCULATING_CONTRIBUTIONS
<<< CALCULATING_CONTRIBUTIONS (31364ms)
>>> GENERATING_INNER_PROOFS
<<< GENERATING_INNER_PROOFS (140385ms)
>>> GENERATE_VADCOP_FINAL_PROOF
<<< GENERATE_VADCOP_FINAL_PROOF (3544ms)
successfully generated and verified proof for the program!
```

That's it. You just ran a program inside a zkVM and generated a cryptographic proof that it executed correctly.

::callout{icon="i-heroicons-information-circle"}
The first run takes a few minutes because it compiles the guest to RISC-V and generates the proof. Subsequent runs with unchanged code are faster.
::

## What just happened

`cargo run --release` ran `host/src/main.rs`, which did four things:

1. **Built the guest** into a RISC-V ELF (via the build script)
2. **Executed** the guest with input `n = 1000`, computing 1000 rounds of SHA-256
3. **Generated** a VADCOP STARK proof of that execution
4. **Verified** the proof locally

The host program is straightforward Rust. Open `host/src/main.rs` to see exactly what it does.

---

## Project structure

The project has two crates: a **guest** and a **host**.

The **guest** (`guest/src/main.rs`) is the program that runs inside the ZisK virtual machine. This is the computation being proved. It runs in an isolated, deterministic environment: no filesystem, no network, no randomness. Given the same input, it always produces the same output. This isolation is what makes it provable.

The **host** (`host/src/main.rs`) is a normal Rust program that runs on your machine. It feeds input to the guest, invokes the prover, and handles the resulting proof. When you ran `cargo run --release`, the host compiled the guest, executed it, generated a proof, and verified it.

The template also includes individual binaries under `host/bin/` (`execute`, `prove`, `compressed`, etc.) that let you run each stage independently during development. You don't need them right now.
