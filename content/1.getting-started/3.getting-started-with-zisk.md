This walkthrough creates a ZisK project, writes a Fibonacci program, compiles it to RISC-V, runs it inside the zkVM, and generates a proof. This is the same example we'll build on throughout the docs.

**Prerequisite:** [Install ZisK](/getting-started/install-linux) and confirm it works:

```bash
cargo-zisk --version
```

## Create a project

The `sdk new` command scaffolds a new ZisK project with a guest and host crate:

```bash
cargo-zisk sdk new fibonacci && cd fibonacci
```

This creates a workspace with the following structure:

```
fibonacci/
├── Cargo.toml
├── guest/
│   ├── Cargo.toml
│   └── src/main.rs
└── host/
    ├── Cargo.toml
    ├── build.rs
    ├── src/main.rs
    └── bin/
        ├── execute.rs
        ├── prove.rs
        ├── compressed.rs
        ├── plonk.rs
        ├── verify-constraints.rs
        └── ziskemu.rs
```

The template comes with a SHA-256 example. We'll replace the guest code with our Fibonacci program.

## Write the Fibonacci guest

Open `guest/src/main.rs` and replace its contents with:

```rust
#![no_main]
ziskos::entrypoint!(main);

use ziskos::io;
use serde::{Serialize, Deserialize};

#[derive(Serialize, Deserialize, Debug)]
struct FibResult {
    n: u32,
    value: u64,
}

fn main() {
    let n: u32 = io::read();
    println!("Guest received n = {}", n);

    let mut a: u64 = 0;
    let mut b: u64 = 1;
    for _ in 0..n {
        let temp = b;
        b = a + b;
        a = temp;
    }

    let result = FibResult { n, value: a };
    println!("Fibonacci({}) = {}", n, a);
    io::commit(&result);
}
```

The guest reads `n` from the host, computes the nth Fibonacci number, and commits the result to the proof's public output. We'll break down the code in the next chapter.

Update `guest/Cargo.toml`:

```toml
[package]
name = "fibonacci-guest"
version = "0.1.0"
edition = "2021"

[dependencies]
ziskos = { git = "https://github.com/0xPolygonHermez/zisk.git" }
serde = { version = "1", features = ["derive"] }
```

## Build the guest

Cross-compile the guest into a RISC-V ELF binary from the `guest/` directory:

```bash
cd guest && cargo-zisk build --release && cd ..
```

## Prepare input

The guest reads `n` as a `u32` via `io::read()`. Create a binary input file with `n = 10`:

```bash
printf '\x0a\x00\x00\x00' > input.bin
```

That's the number 10 encoded as a little-endian `u32` (4 bytes).

## Run in the emulator

Execute the guest without generating any proof:

```bash
cargo-zisk execute \
  -e target/riscv64ima-zisk-zkvm-elf/release/fibonacci-guest \
  -i input.bin -l
```

Expected output:

```
Guest received n = 10
Fibonacci(10) = 55
```

`-l` selects the emulator backend (works on all platforms). On Linux, omit it for the faster ASM backend.

## Verify constraints

Check that the execution trace satisfies all of ZisK's PIL constraints. This catches witness issues early, without the cost of full proving:

```bash
cargo-zisk verify-constraints \
  -e target/riscv64ima-zisk-zkvm-elf/release/fibonacci-guest \
  -i input.bin -l
```

Expected output:

```
[INFO ] CstrVrfy: All global constraints were successfully verified
[INFO ] CstrVrfy: All constraints were verified
```

## Generate a proof

ZisK re-executes the program, builds the full execution trace, and produces a VADCOP STARK proof:

```bash
cargo-zisk prove \
  -e target/riscv64ima-zisk-zkvm-elf/release/fibonacci-guest \
  -i input.bin -o /tmp/fib-proof -l -a
```

`-a` enables aggregation, which produces the final VADCOP proof. Without it, only inner proofs are generated.

Expected output:

```
[INFO ] ✓ Vadcop Final proof was verified
```

Anyone can now verify this proof without re-running the program or knowing the input.
