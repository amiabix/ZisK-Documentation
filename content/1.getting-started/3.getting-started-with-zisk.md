This walkthrough creates a ZisK project, writes a Fibonacci guest program, executes it from a host binary, and generates a proof. This is the same example we'll build on throughout the docs.

**Prerequisite:** [Install ZisK](/getting-started/install-linux) and confirm it works:

```bash
cargo-zisk --version
```

## Create a project

The `sdk new` command scaffolds a new ZisK workspace with a guest crate (the program that runs inside the zkVM) and a host crate (the program that drives execution and proving from outside):

```bash
cargo-zisk sdk new fibonacci && cd fibonacci
```

This creates the following structure:

```
fibonacci/
├── Cargo.toml          # workspace root
├── guest/
│   ├── Cargo.toml
│   └── src/main.rs
└── host/
    ├── Cargo.toml
    ├── build.rs        # compiles guest + generates default input
    ├── src/main.rs     # execute + prove combined
    └── bin/
        ├── execute.rs
        ├── prove.rs
        ├── compressed.rs
        ├── plonk.rs
        ├── verify-constraints.rs
        └── ziskemu.rs
```

The workspace `Cargo.toml` pins both SDK dependencies to the same branch:

```toml
[workspace]
members = ["host", "guest"]
resolver = "2"

[workspace.dependencies]
zisk-sdk = { git = "https://github.com/0xPolygonHermez/zisk.git", branch = "0.16" }
ziskos = { git = "https://github.com/0xPolygonHermez/zisk.git", branch = "0.16" }
```

The template comes with a SHA-256 example. We'll replace the guest code with a Fibonacci program.

## Write the Fibonacci guest

Replace `guest/src/main.rs` with:

```rust
#![no_main]
ziskos::entrypoint!(main);

use ziskos::io;
use serde::{Serialize, Deserialize};

#[derive(Serialize, Deserialize, Debug)]
struct FibResult {
    n: u32,
    value: u64,
}

fn main() {
    let n: u32 = io::read();
    println!("Guest received n = {}", n);

    let mut a: u64 = 0;
    let mut b: u64 = 1;
    for _ in 0..n {
        let temp = b;
        b = a + b;
        a = temp;
    }

    let result = FibResult { n, value: a };
    println!("Fibonacci({}) = {}", n, a);
    io::commit(&result);
}
```

The guest reads `n` from the host via `io::read()`, computes the nth Fibonacci number, and commits the result as public output via `io::commit()`.

Update `guest/Cargo.toml` to use the workspace dependency:

```toml
[package]
name = "guest"
version = "0.1.0"
edition = "2024"

[dependencies]
ziskos = { workspace = true }
serde = { version = "1.0", default-features = false, features = ["derive"] }
```

## Update the host build script

The host's `build.rs` compiles the guest ELF and generates a default input file. Replace `host/build.rs` with:

```rust
use std::path::PathBuf;
use zisk_sdk::{ZiskIO, ZiskStdin, build_program};

fn main() {
    build_program("../guest");

    let n = 10u32;
    let stdin = ZiskStdin::new();
    stdin.write(&n);

    let path = PathBuf::from("tmp/input.bin");
    if let Some(parent) = path.parent() {
        std::fs::create_dir_all(parent).unwrap();
    }
    stdin.save(&path).unwrap();
}
```

This runs at `cargo build` time -- it cross-compiles the guest to a RISC-V ELF and writes a serialized input file with `n = 10`.

## Write the execute binary

Replace `host/bin/execute.rs` with:

```rust
use anyhow::Result;
use serde::{Deserialize, Serialize};
use zisk_sdk::{ZiskStdin, ZiskIO, ElfBinary, ProverClient, include_elf};

pub const ELF: ElfBinary = include_elf!("guest");

#[derive(Serialize, Deserialize, Debug)]
struct FibResult {
    n: u32,
    value: u64,
}

fn main() -> Result<()> {
    let n = 10u32;
    let stdin = ZiskStdin::new();
    stdin.write(&n);

    let client = ProverClient::builder().emu().build().unwrap();
    let (pk, _) = client.setup(&ELF)?;

    let result = client.execute(&pk, stdin)?;
    println!("Cycles: {}", result.get_execution_steps());
    println!("Duration: {:?}", result.get_duration());

    let output: FibResult = result.get_public_values()?;
    println!("Output: {:?}", output);

    Ok(())
}
```

Key points:

- `include_elf!("guest")` embeds the compiled guest ELF at compile time (built by `build.rs`).
- `ZiskStdin` serializes host-side inputs that the guest reads with `io::read()`.
- `ProverClient::builder().emu().build()` creates a client using the emulator backend, which works on all platforms (macOS, Linux). For production on Linux, use `.asm()` instead for significantly faster execution.
- `client.setup(&ELF)` returns a proving key and verifying key for the program.
- `client.execute()` runs the guest and returns the result without generating a proof.
- `result.get_public_values()` deserializes the data the guest committed via `io::commit()`.

## Run it

From the project root:

```bash
cargo run --release --bin execute
```

Expected output:

```
[INFO ] ... PROOF INSTANCES SUMMARY
[INFO ] Zisk | Arith: 1 | Binary: 1 | BinaryAdd: 1 | BinaryExtension: 1 |
       Dma64AlignedMemCpy: 1 | DmaMemCpy: 1 | DmaPrePostMemCpy: 1 |
       InputData: 1 | Main: 1 | Mem: 1 | MemAlign: 1 | Rom: 1 |
       RomData: 1 | SpecifiedRanges: 1 | VirtualTable0: 1 |
       VirtualTable1: 1 | Total global instances: 16
<<< EXECUTE (158ms)

```

The guest ran inside the zkVM, computed `Fibonacci(10) = 55`, and the host read the committed result back as a typed Rust struct.

## Generate a proof

Proving follows the same pattern -- build a `ProverClient`, provide the ELF and input, then call `prove` instead of `execute`. The template includes `host/bin/prove.rs` for this. Run it with:

```bash
cargo run --release --bin prove
```

The official quickstart flow runs each stage from the project root: `execute` -> `verify-constraints` -> `prove` -> `compressed`. The next chapter covers the prove workflow in detail, including proof options, verification, and saving proofs to disk.
