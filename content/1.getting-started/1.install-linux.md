---
title: "Install on Linux"
description: "Install ZisK on Ubuntu 22.04+ (x86_64), the recommended platform with full proving support."
---

Linux is the recommended platform for ZisK. It supports everything: the fast ASM execution backend, GPU-accelerated proving, MPI parallelism, and distributed proving. You'll need Ubuntu 22.04+ on x86_64.

## Dependencies

Make sure [Rust](https://www.rust-lang.org/tools/install) and [Git](https://git-scm.com/) are installed, then install system dependencies:

```bash
sudo apt-get install -y \
  xz-utils jq curl build-essential qemu-system \
  libomp-dev libgmp-dev nlohmann-json3-dev \
  protobuf-compiler uuid-dev libgrpc++-dev \
  libsecp256k1-dev libsodium-dev libpqxx-dev nasm \
  libopenmpi-dev openmpi-bin openmpi-common \
  libclang-dev clang gcc-riscv64-unknown-elf
```

---

## Shared memory

ZisK uses shared memory to exchange data between processes. The system must allow unlimited locked memory:

```bash
ulimit -l
```

If this does not print `unlimited`, edit `/etc/systemd/system.conf`:

```
DefaultLimitMEMLOCK=infinity
```

Then reboot for changes to take effect.

## Install ZisK

### Option 1: Prebuilt binaries (recommended)

```bash
curl https://raw.githubusercontent.com/0xPolygonHermez/zisk/main/ziskup/install.sh | bash
```

During installation, you will be prompted to select a setup option:

| Option | What it installs | Use case |
| --- | --- | --- |
| **Install proving key** (default) | Full proving key | Generate and verify proofs |
| **Install proving key (no constant tree files)** | Proving key without constant trees | Proving key without pre-generated constant trees |
| **Install verify key** | Verification key only | Verify proofs, no generation |
| **None** | Toolchain only | Compile and emulate programs |

Verify it worked:

```bash
rustup toolchain list
```

You should see `zisk` in the list.

```bash
cargo-zisk --version
```

### Option 2: Building from source

**1. Clone the ZisK repository:**

```bash
git clone https://github.com/0xPolygonHermez/zisk.git
cd zisk
```

**2. Build ZisK tools:**

```bash
cargo build --release
```

::callout{icon="i-heroicons-information-circle"}
If you encounter `stddef.h file not found` during compilation on Ubuntu:
```bash
find /usr -name "stddef.h"
export C_INCLUDE_PATH=/usr/lib/gcc/x86_64-linux-gnu/13/include
export CPLUS_INCLUDE_PATH=$C_INCLUDE_PATH
```
Then retry the build.
::

**3. Copy the tools to `~/.zisk/bin`:**

```bash
mkdir -p $HOME/.zisk/bin
LIB_EXT=$([[ "$(uname)" == "Darwin" ]] && echo "dylib" || echo "so")
cp target/release/cargo-zisk \
   target/release/ziskemu \
   target/release/riscv2zisk \
   target/release/zisk-coordinator \
   target/release/zisk-worker \
   target/release/libzisk_witness.$LIB_EXT \
   target/release/libziskclib.a \
   $HOME/.zisk/bin
```

**4. Copy required files for assembly ROM setup (Linux x86_64 only):**

```bash
mkdir -p $HOME/.zisk/zisk/emulator-asm
cp -r ./emulator-asm/src $HOME/.zisk/zisk/emulator-asm
cp ./emulator-asm/Makefile $HOME/.zisk/zisk/emulator-asm
cp -r ./lib-c $HOME/.zisk/zisk
```

**5. Add `~/.zisk/bin` to your PATH:**

```bash
PROFILE=$([[ "$(uname)" == "Darwin" ]] && echo ".zshenv" || echo ".bashrc")
echo >> $HOME/$PROFILE && echo 'export PATH="$PATH:$HOME/.zisk/bin"' >> $HOME/$PROFILE
source $HOME/$PROFILE
```

**6. Install the ZisK Rust toolchain:**

```bash
cargo-zisk sdk install-toolchain
```

::callout{icon="i-heroicons-information-circle"}
To build the toolchain from source instead (requires all [Rust build dependencies](https://github.com/rust-lang/rust/blob/master/INSTALL.md#dependencies)):
```bash
cargo-zisk sdk build-toolchain
```
::

**7. Verify the installation:**

```bash
rustup toolchain list
```

Confirm that `zisk` appears in the list.

#### Building the setup (proving key)

This process takes approximately 45 to 60 minutes. [Node.js](https://nodejs.org/en/download) 20.x+ is required.

**1. Clone required repositories** (in the parent folder of `zisk`):

```bash
git clone https://github.com/0xPolygonHermez/pil2-compiler.git
git clone https://github.com/0xPolygonHermez/pil2-proofman.git
git clone https://github.com/0xPolygonHermez/pil2-proofman-js
```

**2. Install packages:**

```bash
(cd pil2-compiler && npm i)
(cd pil2-proofman-js && npm i)
```

**3. From the `zisk` folder, generate fixed data:**

```bash
cd zisk
cargo run --release --bin arith_frops_fixed_gen
cargo run --release --bin binary_basic_frops_fixed_gen
cargo run --release --bin binary_extension_frops_fixed_gen
```

**4. Compile ZisK PIL:**

```bash
node ../pil2-compiler/src/pil.js \
  pil/zisk.pil \
  -I pil,../pil2-proofman/pil2-components/lib/std/pil,state-machines,precompiles \
  -o pil/zisk.pilout \
  -u tmp/fixed -O fixed-to-file
```

**5. Generate setup data:**

```bash
node ../pil2-proofman-js/src/main_setup.js \
  --stack-size=8192 \
  -a ./pil/zisk.pilout \
  -b build \
  -t ../pil2-proofman/pil2-components/lib/std/pil \
  -u tmp/fixed -r \
  -s ./state-machines/starkstructs.json
```

This generates the `build/provingKey` directory.

**6. Copy the setup to `~/.zisk`:**

```bash
cp -R build/provingKey $HOME/.zisk
```

## Generate constant trees

If you installed the proving key (via prebuilt or from source), run:

```bash
cargo-zisk check-setup -a
```

This generates constant tree files needed for proving.

---

## Updating

```bash
ziskup
```

Flags: `--provingkey`, `--verifykey`, `--nokey` to skip the prompt.

---

## Uninstall

```bash
rustup uninstall zisk
rm -rf $HOME/.zisk
```

Once installed, head to [Quickstart](/getting-started/getting-started-with-zisk) to create your first project and generate a proof.
