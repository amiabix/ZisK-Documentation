ZisK provides three ways to verify proofs: the CLI, `prover.verify()`, and standalone verification functions.

## Three Ways to Verify

| Method | When to use |
| --- | --- |
| CLI (`cargo-zisk verify`) | Quick checks, CI pipelines, command-line workflows |
| SDK (`prover.verify()`) | When you already have a ProverClient in your program |
| Standalone functions | Verification-only services, no ProverClient needed |

All three perform the same checks.

## CLI Verification

```bash
# Verify a VADCOP or compressed proof
cargo-zisk verify -p /tmp/proof-dir/proof.bin

# Verify a PLONK SNARK proof
cargo-zisk verify-snark -p /tmp/proof-dir/snark_proof.bin -k /tmp/proof-dir/verkey.bin
```

Exits 0 on success, non-zero on failure. CI example:

```bash
cargo-zisk prove -e target/riscv64ima-zisk-zkvm-elf/release/my-guest \
    -i input.bin -o /tmp/proof -l -a

cargo-zisk verify -p /tmp/proof/proof.bin

echo "Proof verified successfully"
```

See [CLI Reference](/cli-reference) for all flags.

## SDK Verification

```rust
use anyhow::Result;
use zisk_sdk::{ProverClient, ZiskIO, ZiskStdin, include_elf, ElfBinary};

const GUEST_ELF: ElfBinary = include_elf!("my-guest-crate");

fn main() -> Result<()> {
    let prover = ProverClient::builder().emu().build().unwrap();
    let (pk, vk) = prover.setup(&GUEST_ELF)?;

    let stdin = ZiskStdin::new();
    stdin.write(&42u64);

    let result = prover.prove(&pk, stdin).run()?;

    // Verify the proof against the verification key
    prover.verify(result.get_proof(), result.get_publics(), &vk)?;
    println!("Proof is valid");
    Ok(())
}
```

`verify()` returns a `Result`. The vk is bound to the guest program. A proof for one guest won't verify against a different vk.

## Standalone Verification Functions

For verification without a `ProverClient` (e.g., a verification-only microservice):

```rust
use zisk_sdk::{verify_zisk_proof, verify_zisk_snark_proof};

// Verify a VADCOP or compressed proof
verify_zisk_proof(&zisk_proof, &publics, &program_vk)?;

// Verify a PLONK SNARK proof
verify_zisk_snark_proof(&proof, &publics, &program_vk)?;
```

These create their own verification context. Useful for decoupling proving from verification: one service generates and saves, another loads and verifies:

```rust
use anyhow::Result;
use zisk_sdk::{verify_zisk_proof, ZiskProofWithPublicValues};

fn verify_from_disk(proof_dir: &str) -> Result<()> {
    let loaded = ZiskProofWithPublicValues::load(proof_dir)?;

    // ZiskProofWithPublicValues contains the proof, publics, and program_vk
    verify_zisk_proof(&loaded.proof, &loaded.publics, &loaded.program_vk)?;
    println!("Proof verified successfully");
    Ok(())
}
```

### With Custom Proving Key Paths

If your proving key artifacts are stored in a non-default location, use the `_with_proving_key` variants:

```rust
use std::path::PathBuf;
use zisk_sdk::verify_zisk_proof_with_proving_key;

// VADCOP/compressed with a custom proving key path
verify_zisk_proof_with_proving_key(
    &zisk_proof,
    &publics,
    &program_vk,
    PathBuf::from("/custom/path/provingKey"),
)?;
```

Same as the standard function but with a custom key path.

## Getting the Verification Key

Obtain a vk without a full prover setup (useful for on-chain registration):

```rust
use zisk_sdk::{get_program_vk, include_elf, ElfBinary};

const GUEST_ELF: ElfBinary = include_elf!("my-guest-crate");

let vk = get_program_vk(&GUEST_ELF)?;
```

Same vk as `prover.setup()` without the overhead. Takes anything implementing `ElfBinaryLike`.

## Reading Public Values from a Proof

Read committed public values from the proof result:

```rust
let result = prover.prove(&pk, stdin).run()?;

// Read a typed value from the public outputs
let answer: u64 = result.get_public_values()?;

// Or access the raw public values bytes
let publics = result.get_publics();
```

`get_public_values::<T>()` deserializes into the expected type. Must match what the guest committed. Type mismatch produces a deserialization error.

## What Verification Checks

Verification checks:

1. **Constraint satisfaction.** The trace satisfies all PIL constraints. Every step followed the rules.
2. **Public output binding.** Committed outputs match the execution. Cannot be forged.
3. **Program binding.** Proof is bound to the correct vk. Program A's proof won't verify against program B's vk.
4. **Integrity.** Polynomial commitments, evaluations, and Fiat-Shamir challenges are consistent.

A proof either verifies completely or fails. No partial success.

## Verification Errors

Common causes of verification failure:

- **Wrong ELF.** You are verifying against a different guest binary than the one used to generate the proof. Recompile and regenerate.
- **Corrupted proof.** The proof file was truncated or modified in transit. Regenerate or re-transfer.
- **Mismatched proof type.** You used `verify_zisk_proof()` for a PLONK proof or `verify_zisk_snark_proof()` for a VADCOP proof. Match the function to the proof type.
- **Wrong proving key path.** The verification function cannot find the key artifacts. Check the path.

## Full Example

```rust
use anyhow::Result;
use zisk_sdk::{
    ProverClient, ZiskIO, ZiskStdin, ZiskProofWithPublicValues,
    verify_zisk_proof, include_elf, ElfBinary,
};

const GUEST_ELF: ElfBinary = include_elf!("my-guest-crate");

fn main() -> Result<()> {
    // --- Proving side ---
    let prover = ProverClient::builder().emu().build().unwrap();
    let (pk, vk) = prover.setup(&GUEST_ELF)?;

    let stdin = ZiskStdin::new();
    stdin.write(&100u32);

    let result = prover.prove(&pk, stdin).run()?;

    // Quick local verification
    prover.verify(result.get_proof(), result.get_publics(), &vk)?;

    // Save to disk
    result.save_proof_with_publics("/tmp/my-proof")?;
    println!("Proof saved");

    // --- Verification side (could be a separate process) ---
    let loaded = ZiskProofWithPublicValues::load("/tmp/my-proof")?;
    verify_zisk_proof(&loaded.proof, &loaded.publics, &loaded.program_vk)?;
    println!("Standalone verification passed");

    // Read the public values
    let output: u32 = result.get_public_values()?;
    println!("Verified output: {}", output);
    Ok(())
}
```
