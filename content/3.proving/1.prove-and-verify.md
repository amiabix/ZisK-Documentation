None of the previous steps produced a proof. A proof convinces any verifier that the guest ran correctly and produced the committed outputs, without re-execution.

## Generate a VADCOP STARK proof

Create `host/bin/prove.rs`:

```rust
use anyhow::Result;
use zisk_sdk::{
    include_elf, ElfBinary, ProofOpts, ProverClient,
    ZiskIO, ZiskProof, ZiskProofWithPublicValues, ZiskPublics, ZiskStdin,
};

const ELF: ElfBinary = include_elf!("guest");

fn main() -> Result<()> {
    let stdin = ZiskStdin::new();
    stdin.write(&10u32);

    let client = ProverClient::builder().emu().build().unwrap();
    let (pk, _) = client.setup(&ELF)?;

    let proof_opts = ProofOpts::default().minimal_memory();
    let result = client.prove(&pk, stdin).with_proof_options(proof_opts).run()?;
    println!("Proof generated!");

    // Verify using the VK embedded in the result
    client.verify(result.get_proof(), result.get_publics(), result.get_program_vk())?;
    println!("Proof verified!");

    Ok(())
}
```

The key difference from execute: `client.prove(&pk, stdin)` returns a `ProveBuilder`, and calling `.run()` generates a proof. `verify()` takes three arguments: proof, public values, and verification key.

`ProofOpts::default().minimal_memory()` configures the prover to reduce memory usage -- use this unless you have a specific reason not to.

Without a modifier (`.compressed()` or `.plonk()`), proving produces a **VADCOP STARK**, ZisK's native proof format.

The examples use `.emu()` which works on all platforms. For production proving on Linux, use `.asm()` with `.base_port(54321)` instead:

```rust
let client = ProverClient::builder().asm().base_port(54321).build().unwrap();
```

Run it:

```bash
cargo run --release --bin prove
```

## Three proof types

ZisK supports three proof types, each building on the previous:

| Type | What it is | When to use it |
| --- | --- | --- |
| **VADCOP STARK** | The raw proof from ZisK's constraint system | Development and testing |
| **Compressed STARK** | STARK wrapped in a recursive proof | Off-chain verification where size matters |
| **PLONK SNARK** | STARK compressed to a small SNARK | On-chain verification on Ethereum |

VADCOP STARK is the default (no modifier). For many use cases (off-chain verification, internal pipelines, development) it's all you need. When proof size matters, compress. When you need on-chain verification, go to PLONK.

## Compressed STARK proofs

A compressed proof wraps the VADCOP STARK in a recursive proof, reducing its size significantly.

**One-step** -- the `ProveBuilder` handles everything:

```rust
let result = client.prove(&pk, stdin)
    .with_proof_options(proof_opts)
    .compressed()
    .run()?;

client.verify(result.get_proof(), result.get_publics(), result.get_program_vk())?;
```

**Two-step** -- prove VADCOP first, then compress separately. Useful when you want to inspect or store the intermediate VADCOP proof:

```rust
let vadcop_result = client.prove(&pk, stdin).with_proof_options(proof_opts).run()?;
let compressed_result = client.compress(
    vadcop_result.get_proof(),
    vadcop_result.get_publics(),
    &vkey,
)?;

client.verify(
    compressed_result.get_proof(),
    compressed_result.get_publics(),
    compressed_result.get_program_vk(),
)?;
```

## PLONK SNARK proofs

A PLONK SNARK compresses the proof to a small constant size for on-chain verification. It requires `.snark()` on the builder to initialize SNARK infrastructure:

```rust
let client = ProverClient::builder().emu().snark().build().unwrap();
let (pk, vkey) = client.setup(&ELF)?;
```

**One-step** -- `.plonk()` on the prove call selects the PLONK backend:

```rust
let snark_proof = client.prove(&pk, stdin).plonk().run()?;

client.verify(snark_proof.get_proof(), snark_proof.get_publics(), &vkey)?;
```

**Two-step** -- prove VADCOP first, then convert to SNARK with `prove_snark()`:

```rust
let vadcop_result = client.prove(&pk, stdin).run()?;
let snark_proof = client.prove_snark(
    &vadcop_result.get_proof(),
    &vadcop_result.get_publics(),
    &vkey,
)?;

client.verify(snark_proof.get_proof(), snark_proof.get_publics(), &vkey)?;
```

On-chain verification is covered in a [later chapter](/proving/onchain-verification).

## Saving and loading proofs

Save a proof (with or without public values) for later use:

```rust
// Save proof + publics together
result.save_proof_with_publics("tmp/proof_with_publics.bin")?;

// Save proof alone
result.get_proof().save("tmp/proof.bin")?;
```

Load and verify from disk:

```rust
// Load proof alone -- you must reconstruct publics separately
let proof = ZiskProof::load("tmp/proof.bin")?;
let vk = client.vk(&ELF)?;
let publics = ZiskPublics::write(&output)?;
client.verify(&proof, &publics, &vk)?;

// Load proof + publics together
let proof_with_publics = ZiskProofWithPublicValues::load("tmp/proof_with_publics.bin")?;
client.verify(&proof_with_publics.proof, &proof_with_publics.publics, &vk)?;
```

`client.vk(&ELF)` derives the verification key from the ELF binary -- use this when you don't have the VK from `setup()`.

## Complete example

A prove binary that generates a VADCOP STARK proof, verifies it, reads public values, and saves to disk:

```rust
use anyhow::Result;
use serde::{Deserialize, Serialize};
use zisk_sdk::{
    include_elf, ElfBinary, ProofOpts, ProverClient,
    ZiskIO, ZiskProofWithPublicValues, ZiskStdin,
};

#[derive(Serialize, Deserialize, Debug)]
struct FibResult {
    n: u32,
    value: u64,
}

const ELF: ElfBinary = include_elf!("guest");

fn main() -> Result<()> {
    let stdin = ZiskStdin::new();
    stdin.write(&10u32);

    let client = ProverClient::builder().emu().build().unwrap();
    let (pk, _) = client.setup(&ELF)?;

    let proof_opts = ProofOpts::default().minimal_memory();
    let result = client.prove(&pk, stdin).with_proof_options(proof_opts).run()?;
    println!("Proof generated!");

    client.verify(result.get_proof(), result.get_publics(), result.get_program_vk())?;
    println!("Proof verified!");

    let fib: FibResult = result.get_public_values()?;
    println!("Proven result: Fibonacci({}) = {}", fib.n, fib.value);

    result.save_proof_with_publics("tmp/fibonacci_proof_with_publics.bin")?;
    println!("Proof saved!");

    // Reload and verify from disk
    let proof = ZiskProofWithPublicValues::load("tmp/fibonacci_proof_with_publics.bin")?;
    let vk = client.vk(&ELF)?;
    client.verify(&proof.proof, &proof.publics, &vk)?;
    println!("Loaded proof verified!");

    Ok(())
}
```
