None of the previous steps produced a proof. A proof convinces any verifier that the guest ran correctly and produced the committed outputs, without re-execution.

## Generate a VADCOP STARK proof

Create `host/bin/prove.rs`:

```rust
use anyhow::Result;
use zisk_sdk::{include_elf, ElfBinary, ProverClient, ZiskIO, ZiskStdin};

const ELF: ElfBinary = include_elf!("fibonacci-guest");

fn main() -> Result<()> {
    let stdin = ZiskStdin::new();
    stdin.write(&10u32);

    let client = ProverClient::builder().emu().build().unwrap();
    let (pk, vk) = client.setup(&ELF)?;

    let result = client.prove(&pk, stdin).run()?;
    println!("Proof generated successfully!");

    client.verify(result.get_proof(), result.get_publics(), &vk)?;
    println!("Proof verified!");

    Ok(())
}
```

The key difference from execute: `client.prove(&pk, stdin).run()` generates a proof, and `client.verify()` checks it.

`prove()` returns a `ProveBuilder`. Without a modifier (`.compressed()` or `.plonk()`), it produces a **VADCOP STARK**, ZisK's native proof format. `verify()` takes three arguments: proof, public values, and verification key.

Run it:

```bash
cargo run --release --bin prove
```

## Three proof types

ZisK supports three proof types, each building on the previous:

| Type | What it is | When to use it |
| --- | --- | --- |
| **VADCOP STARK** | The raw proof from ZisK's constraint system | Development and testing |
| **Compressed STARK** | STARK wrapped in a recursive proof | Off-chain verification where size matters |
| **PLONK SNARK** | STARK compressed to a small SNARK | On-chain verification on Ethereum |

VADCOP STARK is the default (no modifier). For many use cases (off-chain verification, internal pipelines, development) it's all you need. When proof size matters, compress. When you need on-chain verification, go to PLONK.

## Compressed STARK proofs

A compressed proof wraps the VADCOP STARK in a recursive proof, reducing its size significantly:

```rust
let result = client.prove(&pk, stdin)
    .compressed()
    .run()?;
```

`.compressed()` adds a recursive compression step. The resulting proof is smaller but verified the same way:

```rust
client.verify(result.get_proof(), result.get_publics(), &vk)?;
```

## PLONK SNARK proofs

A PLONK SNARK compresses the proof to a small constant size for on-chain verification. It requires a different prover configuration:

```rust
let client = ProverClient::builder().emu().snark().build().unwrap();
let (pk, vk) = client.setup(&ELF)?;

let result = client.prove(&pk, stdin)
    .plonk()
    .run()?;

println!("PLONK proof generated!");
client.verify(result.get_proof(), result.get_publics(), &vk)?;
println!("PLONK proof verified!");
```

Two changes: `.snark()` on the builder initializes SNARK infrastructure (must be before `.build()`), and `.plonk()` on the prove call selects the PLONK backend. On-chain verification is covered in a [later chapter](/proving/onchain-verification).

## Saving proofs to disk

You can save any proof to disk for later use:

```rust
result.save_proof_with_publics("/tmp/fibonacci-proof")?;
```

## Complete example

A prove binary that generates a VADCOP STARK proof, verifies it, reads the public values, and saves it:

```rust
use anyhow::Result;
use zisk_sdk::{include_elf, ElfBinary, ProverClient, ZiskIO, ZiskStdin};
use serde::{Serialize, Deserialize};

#[derive(Serialize, Deserialize, Debug)]
struct FibResult {
    n: u32,
    value: u64,
}

const ELF: ElfBinary = include_elf!("fibonacci-guest");

fn main() -> Result<()> {
    let stdin = ZiskStdin::new();
    stdin.write(&10u32);

    let client = ProverClient::builder().emu().build().unwrap();
    let (pk, vk) = client.setup(&ELF)?;

    let result = client.prove(&pk, stdin).run()?;
    println!("Proof generated!");

    client.verify(result.get_proof(), result.get_publics(), &vk)?;
    println!("Proof verified!");

    let fib: FibResult = result.get_public_values()?;
    println!("Proven result: Fibonacci({}) = {}", fib.n, fib.value);

    result.save_proof_with_publics("/tmp/fibonacci-proof")?;
    println!("Proof saved to /tmp/fibonacci-proof");

    Ok(())
}
```
