---
title: Precompiles
description: Native VM instructions for SHA-256, Keccak, elliptic curve arithmetic, and other expensive operations.
---

Without precompiles, a SHA-256 compression expands into thousands of RISC-V instructions, each a row in the execution trace. Precompiles execute these operations as single native VM instructions, producing the same result with a dramatically smaller trace.

## How precompiles work

On a precompile call, ZisK reads inputs from guest memory, executes natively, writes the result back, and records a single trace entry.

No code changes required. Standard Rust crates are automatically routed through native implementations when compiled for the ZisK target.

## Using precompiles

Most precompiles activate automatically through standard Rust crates:

```rust
use sha2::{Sha256, Digest};

fn main() {
    let data = b"hello, world";
    let hash = Sha256::digest(data);
    // This executes through the sha256_compress precompile.
    // The result is identical to running SHA-256 in software,
    // but the execution trace is much smaller.
    println!("hash: {:x}", hash);
}
```

Keccak hashing and elliptic curve crates (secp256k1, secp256r1, BN254, BLS12-381) are similarly routed through their corresponding precompiles on the ZisK target.

## DMA macros

ZisK provides macros for DMA precompiles when manipulating large memory blocks:

```rust
use ziskos::{dma_memcpy, dma_memset, dma_memmove, dma_memcmp};

fn main() {
    let src = [1u8; 1024];
    let mut dst = [0u8; 1024];

    // Copy 1024 bytes from src to dst
    dma_memcpy(dst.as_mut_ptr(), src.as_ptr(), 1024);

    // Set 512 bytes to zero
    dma_memset(dst.as_mut_ptr(), 0, 512);
}
```

Behave like their C standard library counterparts but execute through DMA precompiles, compressing operations into a small number of trace rows.

## Impact on proving

Precompiles reduce trace rows, directly reducing proving time. The impact is most significant for heavy cryptography. ECDSA verification, for example, involves many field operations that each collapse to a single native operation.

If proving is slow, check whether your most expensive operations have precompile coverage. Using standard Rust crates is usually enough. ZisK handles routing automatically.
