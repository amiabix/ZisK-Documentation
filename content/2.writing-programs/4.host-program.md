The CLI works for quick checks, but requires hand-encoding inputs as raw bytes and doesn't give you programmatic access to results. The host program solves both problems.

The host is a normal Rust binary that prepares input for the guest, drives execution or proving, and reads the guest's committed outputs as typed values.

## The host Cargo.toml

`host/Cargo.toml`:

```toml
[package]
name = "host"
version = "0.1.0"
edition = "2024"

[dependencies]
zisk-sdk = { workspace = true }
anyhow = "1.0"
serde = { version = "1.0", default-features = false, features = ["derive"] }

[build-dependencies]
zisk-sdk = { workspace = true }

[features]
default = []
gpu = ["zisk-sdk/gpu"]

[[bin]]
name = "prove"
path = "bin/prove.rs"

[[bin]]
name = "execute"
path = "bin/execute.rs"

[[bin]]
name = "plonk"
path = "bin/plonk.rs"

[[bin]]
name = "compressed"
path = "bin/compressed.rs"

[[bin]]
name = "ziskemu"
path = "bin/ziskemu.rs"
```

Dependencies use `workspace = true` -- the actual versions and git source are declared once in the workspace-level `Cargo.toml`. Each `[[bin]]` entry maps to a separate binary under `host/bin/`.

::callout{icon="i-heroicons-information-circle"}
On macOS, add `features = ["disable_distributed"]` to both `zisk-sdk` entries in the **workspace** `Cargo.toml`. Distributed proving relies on MPI, which is not available on macOS.
::

## The build script

The host needs access to the guest's compiled ELF binary. `host/build.rs` compiles the guest and also pre-generates a default input file:

```rust
use std::path::PathBuf;
use zisk_sdk::{ZiskIO, ZiskStdin, build_program};

fn main() {
    build_program("../guest");
    let n = 1000u32;
    let stdin_save = ZiskStdin::new();
    stdin_save.write(&n);
    let path = PathBuf::from("tmp/input.bin");
    if let Some(parent) = path.parent() {
        std::fs::create_dir_all(parent).unwrap();
    }
    stdin_save.save(&path).unwrap();
}
```

`build_program("../guest")` compiles the guest (if needed) and makes the resulting ELF available to the `include_elf!` macro. The rest serializes a default input value and saves it to `tmp/input.bin` -- useful for CLI-based runs and testing.

## Write the execute binary

Create `host/bin/execute.rs`. This runs the guest without generating a proof:

```rust
use anyhow::Result;
use serde::{Deserialize, Serialize};
use zisk_sdk::{ZiskStdin, ZiskIO, ElfBinary, ProverClient, include_elf};

pub const ELF: ElfBinary = include_elf!("guest");

#[derive(Serialize, Deserialize, Debug)]
struct FibResult {
    n: u32,
    value: u64,
}

fn main() -> Result<()> {
    let n = 10u32;
    let stdin = ZiskStdin::new();
    stdin.write(&n);

    let client = ProverClient::builder().emu().build().unwrap();
    let (pk, _) = client.setup(&ELF)?;

    let result = client.execute(&pk, stdin.clone())?;
    println!("Cycles: {}", result.get_execution_steps());
    println!("Duration: {:?}", result.get_duration());

    let output: FibResult = result.get_public_values()?;
    println!("Fibonacci({}) = {}", output.n, output.value);
    Ok(())
}
```

Key points:

- **`include_elf!("guest")`** embeds the compiled guest ELF into the host binary. The argument is the guest crate's package name from `guest/Cargo.toml`.
- **`ZiskStdin`** is the input buffer. `.write()` serializes and appends. Uses interior mutability, so no `mut` needed. Values are read in the same order by the guest.
- **`ProverClient::builder().emu().build()`** creates the prover with the emulator backend. `.emu()` works on all platforms. For production on Linux, use `.asm()` instead -- it runs natively and is significantly faster.
- **`.setup(&ELF)`** returns the proving key (`pk`) and verification key (`vk`).
- **`.execute(&pk, stdin.clone())`** runs the guest without generating a proof.
- **`.get_public_values::<T>()`** deserializes the guest's committed output into the requested type.
- **`.get_execution_steps()`** and **`.get_duration()`** report cycle count and wall-clock time for the execution.

## Run it

From the project root:

```bash
cargo run --release --bin execute
```

Expected output:

```
Cycles: ...
Duration: ...
Fibonacci(10) = 55
```

We've only executed the program so far -- no proof. The next section covers proving and verification.
