The CLI works for quick checks, but requires hand-encoding inputs as raw bytes and doesn't give you programmatic access to results. The host program solves both problems.

The host is a normal Rust binary that prepares input for the guest, drives execution or proving, and reads the guest's committed outputs as typed values.

## The build script

The host needs access to the guest's compiled ELF binary. The scaffolded `host/build.rs` uses a ZisK helper that compiles the guest automatically and embeds the binary into the host at build time:

```rust
fn main() {
    zisk_sdk::build_program("../guest");
}
```

Every time you build the host, this compiles the guest (if needed) and makes the resulting ELF available to the `include_elf!` macro.

## The host Cargo.toml

`host/Cargo.toml`:

```toml
[package]
name = "fibonacci-host"
version = "0.1.0"
edition = "2021"

[build-dependencies]
zisk-sdk = { git = "https://github.com/0xPolygonHermez/zisk.git" }

[dependencies]
zisk-sdk = { git = "https://github.com/0xPolygonHermez/zisk.git" }
serde = { version = "1", features = ["derive"] }
bincode = "1"
anyhow = "1"
```

::callout{icon="i-heroicons-information-circle"}
On macOS, add `features = ["disable_distributed"]` to both `zisk-sdk` entries. Distributed proving relies on MPI, which is not available on macOS.
```toml
[build-dependencies]
zisk-sdk = { git = "https://github.com/0xPolygonHermez/zisk.git", features = ["disable_distributed"] }

[dependencies]
zisk-sdk = { git = "https://github.com/0xPolygonHermez/zisk.git", features = ["disable_distributed"] }
```
::

`zisk-sdk` appears in both `[build-dependencies]` (for the build script) and `[dependencies]` (for host code).

## Write the execute binary

Create `host/bin/execute.rs`. This runs the guest without generating a proof:

```rust
use zisk_sdk::{include_elf, ElfBinary, ProverClient, ZiskIO, ZiskStdin};
use serde::{Serialize, Deserialize};
use anyhow::Result;

#[derive(Serialize, Deserialize, Debug)]
struct FibResult {
    n: u32,
    value: u64,
}

const ELF: ElfBinary = include_elf!("fibonacci-guest");

fn main() -> Result<()> {
    let stdin = ZiskStdin::new();
    stdin.write(&10u32);

    let prover = ProverClient::builder().emu().build().unwrap();
    let (pk, vk) = prover.setup(&ELF)?;
    let execution_result = prover.execute(&pk, stdin)?;

    let result: FibResult = execution_result.get_public_values::<FibResult>()?;
    println!("Fibonacci({}) = {}", result.n, result.value);

    Ok(())
}
```

Key points:

- **`include_elf!("fibonacci-guest")`** embeds the compiled guest ELF into the host binary. The argument is the guest crate's package name from `guest/Cargo.toml`.
- **`ZiskStdin`** is the input buffer. `.write()` serializes with bincode and appends. Uses interior mutability, so no `mut` needed. Values are read in the same order by the guest.
- **`ProverClient::builder().emu().build()`** creates the prover with the emulator backend. `.emu()` works on all platforms; use `.asm()` on Linux for production.
- **`.setup(&ELF)`** returns the proving key (`pk`) and verification key (`vk`).
- **`.execute(&pk, stdin)`** runs the guest without generating a proof. Takes `stdin` by value.
- **`.get_public_values::<T>()`** deserializes the guest's committed output into the requested type.

## Run it

From the workspace root:

```bash
cargo run --release --bin execute
```

Expected output:

```
Fibonacci(10) = 55
```

We've only executed the program so far, no proof. The next section covers proving and verification.
