Compile the guest to RISC-V, run it in the emulator, verify constraints, and generate a proof.

## Build the guest

From the `guest/` directory:

```bash
cd guest && cargo-zisk build --release && cd ..
```

This cross-compiles the guest into a RISC-V ELF binary at `target/riscv64ima-zisk-zkvm-elf/release/fibonacci-guest`.

## Prepare input

The guest reads `n` as a `u32` via `io::read()`. Create a binary input file with `n = 10`:

```bash
printf '\x0a\x00\x00\x00' > input.bin
```

That's the number 10 encoded as a little-endian `u32` (4 bytes).

## Run in the emulator

```bash
cargo-zisk execute \
  -e target/riscv64ima-zisk-zkvm-elf/release/fibonacci-guest \
  -i input.bin -l
```

`-l` selects the emulator backend (works on all platforms). On Linux, omit it for the faster ASM backend.

If the program requires a large number of ZisK steps, you might encounter an `EmulationNoCompleted` error. Increase the step limit with the `-n` (`--max-steps`) flag:

```bash
ziskemu -e target/riscv64ima-zisk-zkvm-elf/release/fibonacci-guest -i input.bin -n 10000000000
```

## Metrics and Statistics

### Performance Metrics

Get performance metrics for a program execution using the `-m` (`--log-metrics`) flag:

```bash
ziskemu -e target/riscv64ima-zisk-zkvm-elf/release/fibonacci-guest -i input.bin -m
```

Output includes execution time, throughput, and clock cycles per step:

```
process_rom() steps=85309 duration=0.0009 tp=89.8565 Msteps/s freq=3051.0000 33.9542 clocks/step
...
```

## Verify constraints

Check that the execution trace satisfies all PIL constraints before spending time on a full proof:

```bash
cargo-zisk verify-constraints \
  -e target/riscv64ima-zisk-zkvm-elf/release/fibonacci-guest \
  -i input.bin -l
```

Expected output:

```
[INFO ] CstrVrfy: All global constraints were successfully verified
[INFO ] CstrVrfy: All constraints were verified
```

If a constraint fails, there's a bug in the trace. Better to catch it here than after a full prove.

## Program Setup

Before generating a proof, you need to generate the program setup files. This must be done the first time after building the ELF, or any time it changes:

```bash
cargo-zisk rom-setup -e target/riscv64ima-zisk-zkvm-elf/release/fibonacci-guest
```

The setup files are generated in the cache directory at `$HOME/.zisk`. To clear the cache:

```bash
cargo-zisk clean
```

## Generate a proof

```bash
cargo-zisk prove \
  -e target/riscv64ima-zisk-zkvm-elf/release/fibonacci-guest \
  -i input.bin -o /tmp/fib-proof -l -a -y
```

- `-a` enables aggregation, producing the final VADCOP proof. Without it, only inner proofs are generated.
- `-y` verifies the proof immediately after generation.

Expected output:

```
[INFO ] ProofMan:     ✓ Vadcop Final proof was verified
[INFO ] ProofMan: Proofs generated successfully
```

To verify a proof separately:

```bash
cargo-zisk verify -p /tmp/fib-proof/vadcop_final_proof.bin
```

See the [CLI Reference](/cli-reference) for all flags.
