---
title: "Multiple Programs"
description: "How to use multiple guest programs in a single host, with separate setup, prove, and verify cycles."
---

A single host can work with multiple guest programs. Each guest gets its own ELF binary, its own program keys, and its own prove/verify cycle. The `ProverClient` singleton is reused across all of them.

::

## Project structure

Add a second guest crate alongside the first:

```
my-project/
  guest/
    Cargo.toml        # name = "fibonacci-guest"
    src/main.rs
  guest_2/
    Cargo.toml        # name = "fibonacci-guest-2"
    src/main.rs
  host/
    build.rs
    Cargo.toml
    src/main.rs
```

Each guest crate is a standalone ZisK program with its own `Cargo.toml` and entry point.

## Build both guests

In `host/build.rs`, call `build_program` for each guest crate:

```rust
fn main() {
    zisk_sdk::build_program("../guest");
    zisk_sdk::build_program("../guest_2");
}
```

Each call cross-compiles the guest to a RISC-V ELF and sets the `ZISK_ELF_<name>` environment variable that `include_elf!` reads at compile time.

## Host program

In `host/src/main.rs`, include both ELFs and set up each one separately:

```rust
use anyhow::Result;
use zisk_sdk::{include_elf, ElfBinary, ProofOpts, ProverClient, ZiskIO, ZiskStdin};

pub const ELF: ElfBinary = include_elf!("fibonacci-guest");
pub const ELF2: ElfBinary = include_elf!("fibonacci-guest-2");

fn main() -> Result<()> {
    let client = ProverClient::builder().emu().build().unwrap();

    // Set up both programs (each gets its own keys)
    let (pk, vk) = client.setup(&ELF)?;
    let (pk2, vk2) = client.setup(&ELF2)?;

    // --- First program ---
    let stdin = ZiskStdin::new();
    stdin.write(&1000u32);

    let result = client.execute(&pk, stdin.clone())?;
    println!("Program 1: {} cycles", result.get_execution_steps());

    let proof_opts = ProofOpts::default().minimal_memory();
    let proof_result = client.prove(&pk, stdin)
        .with_proof_options(proof_opts)
        .run()?;

    client.verify(proof_result.get_proof(), proof_result.get_publics(), &vk)?;
    println!("Program 1: proof verified");

    // --- Second program ---
    let stdin2 = ZiskStdin::new();
    stdin2.write(&2000u32);

    let result2 = client.execute(&pk2, stdin2.clone())?;
    println!("Program 2: {} cycles", result2.get_execution_steps());

    let proof_opts2 = ProofOpts::default().minimal_memory();
    let proof_result2 = client.prove(&pk2, stdin2)
        .with_proof_options(proof_opts2)
        .run()?;

    client.verify(proof_result2.get_proof(), proof_result2.get_publics(), &vk2)?;
    println!("Program 2: proof verified");

    Ok(())
}
```

## Key points

- **One `ProverClient`, multiple programs.** The client is a singleton. Call `.setup()` with each ELF to get separate program keys. Do not create multiple `ProverClient` instances.
- **Each `include_elf!` name must match the guest's `package.name`** in its `Cargo.toml`. The macro reads the environment variable `ZISK_ELF_<name>` set by `build_program`.
- **Proofs are independent.** Each program produces its own proof with its own public values. There is no cross-program proof composition.
- **Execution is sequential.** Each prove call blocks until complete. Run them one after another.
